# SaaS 相关

## SaaS 功能和独立部署一样吗？

一样。课表、倒计时、天气、自动任务、备份，功能完全一致。

## 我的数据安全吗？

每个学校有独立的 namespace，互相看不到对方的数据。每个用户的密码加密保存在数据库中，使用 [bcrypt](/glossary/bcrypt) 算法加密，不论是我们还是黑客都无法反推出您的密码。数据库会定期自动备份，您的数据很安全！

## 以后想迁移到独立部署怎么办？

管理后台有个「完整导出」，导出 JSON 文件，然后在独立部署的后端导入就行。

## 网络有什么要求？

简而言之：客户端和管理后台要能上网。学校网络如果有限制，放行 `*.getastra.cn` 和 `hubproxy.khbit.cn` 这两个域名即可。

如果您的学校需要更严格的网路控制，具体而言，您需要放行：

| 域名                | 用途     | 备注                     |
| ------------------- | -------- | ------------------------ |
| `i.getastra.cn`     | 配置管理 |                          |
| `hubproxy.khbit.cn` | 自动更新 |                          |
| `YOURS.getastra.cn` | 配置获取 | `YOURS` 为您申请的子域名 |

## SaaS 模式需要自己维护服务器吗？

不需要。服务器、数据库、HTTPS 证书等等都由我们维护。

## SaaS 模式的客户端地址怎么填？

填你注册时拿到的子域名，类似 `myschool.getastra.cn`，不要带协议前缀。

## SaaS 模式支持自定义域名吗？

不支持。只能使用 `*.getastra.cn` 的子域名。

SaaS 模式正是为了解决公网环境下安全配置难的问题，一部分安全设置依赖于 [Cloudflare WAF](/glossary/cloudflare)。使用自定义域名会损失这部分保护，降低安全性。

## SaaS 模式能只借用子域名吗？

不可以。

因为我们无法确定您运行的是否是受信任的服务端实现，也无法确定您的服务端版本是否匹配最新的防护规则。
