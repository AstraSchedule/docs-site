# 管理端相关

## 管理端显示白屏？

1. 打开浏览器开发者工具（F12），查看 Console 和 Network 标签
2. 如果请求后端 API 失败，检查：
   - 后端是否运行
   - `src/global.js` 中 `APISRV` 地址是否正确（必须与部署时配置一致）
3. 如果是 Cloudflare Pages 部署，确认构建成功且 `APISRV` 地址包含了协议（`https://`）
4. 尝试清除浏览器缓存（Ctrl+Shift+Delete）

## 保存配置失败？

1. 返回 401：认证失败。检查 JWT Token 是否过期，或写操作的密码确认是否正确。
2. 返回 403：WAF 拦截。按部署相关章节的方式排查 Cloudflare WAF 规则。
3. 返回 500：后端内部错误。查看后端日志定位具体原因（可能是数据库写入失败或数据格式不对）。
4. 请求超时：函数计算冷启动或其他网络问题，稍后重试。
5. 网络错误（Failed to fetch）：后端未运行或网络不通。

## 管理端登录密码忘记了怎么办？

管理端使用 JWT 登录：用户名 + 密码登录后拿到 Token。密码存在数据库 `users` 表（bcrypt 哈希），**不是** `config.toml` 里的 `secret.token`。

默认管理员账号：`admin` / `admin`（首次登录会要求修改密码）。

### 还记得管理员账号

1. 用管理员账号登录
2. 在「用户管理」里重置对应用户的密码

### 管理员密码也忘了

需要能访问数据库或后端运行环境：

1. **SQLite**：用工具打开库文件，查看/更新 `users` 表对应行的密码哈希（需自行生成 bcrypt）
2. **MySQL**：同上，更新 `users` 表
3. 或清空用户表后重启后端，让启动逻辑重建默认 `admin`/`admin`（会丢失其他用户，慎用）

`secret.token` 只用于 JWT 签名。改它会导致已签发 Token 全部失效，**不会**恢复登录密码。

用户账号、角色可通过管理端用户管理页面维护（需 admin 角色）。
