# 最佳实践（Best Practices）

## 通俗解释

最佳实践就是"业内公认的做法"。不是法律，没人强制你遵守，但大家都这么做，因为踩过坑、试过错，最后发现这样做最靠谱。

本文档几乎所有部分都是在提供“最佳实践”，而不是在说明可能的所有方法，因为可能的方法太多了。

## 专业解释

最佳实践是在特定领域内，经过实践验证的、被广泛认可的高效、安全、可维护的方法或流程。

特点：
- **经验总结**：来自实际项目中的成功和失败经验
- **行业共识**：被多数从业者和权威机构认可
- **非强制性**：是建议，不是标准或规范
- **持续演进**：随技术发展不断更新

AstraSchedule 文档中的最佳实践：
- 密码存储使用 bcrypt 而非明文或 MD5
- 配置文件不要提交到版本控制
- API 写操作需要密码二次确认
- 多表操作使用事务保证一致性
- 外网部署使用 HTTPS 和 WAF
