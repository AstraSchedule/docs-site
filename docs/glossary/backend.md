# 后端（Backend）

## 通俗解释

后端就是服务器上运行的程序，用户看不到它。它处理数据、执行业务逻辑、返回结果给前端。比如你提交一个表单，后端负责保存到数据库。

AstraSchedule 的后端是用 Go 语言写的，负责存储课表数据、处理 API 请求。

## 专业解释

后端是服务器端的应用程序，负责处理业务逻辑、数据存储和 API 服务。

核心职责：
- **API 服务**：接收请求，返回响应
- **业务逻辑**：处理数据、执行规则
- **数据存储**：读写数据库
- **认证授权**：验证用户身份，控制访问权限

AstraSchedule 的后端包括：
| 组件 | 技术栈 | 用途 |
|------|--------|------|
| usr-backend | Go + Gin + GORM | 核心 API 服务 |
| sys-backend | Go + Gin | 系统管理 |

后端通常不直接与用户交互，而是通过 API 为前端提供数据和服务。
