# SaaS（软件即服务）

## 通俗解释

SaaS 就是"租软件用"。不用买服务器、不用装软件、不用维护，注册账号登录就能用。像用邮箱、用网盘一样，数据存在服务商那边，你只管用。

AstraSchedule 提供 SaaS 模式，注册后拿到子域名，装上客户端就能用，不用自己部署服务器。

## 专业解释

SaaS（Software as a Service）是一种软件交付模式，用户通过浏览器或客户端访问云端应用，无需本地安装和维护。

特点：
- **无需部署**：服务商负责服务器、数据库、运维
- **按需付费**：通常按用户数或使用量收费
- **自动更新**：新版本自动生效，用户无需操作
- **多租户架构**：多个用户共享基础设施，数据隔离

与传统部署对比：
| | SaaS | 传统部署 |
|---|---|---|
| 服务器 | 服务商提供 | 自己购买和维护 |
| 数据存储 | 服务商托管 | 自己控制 |
| 运维 | 服务商负责 | 自己负责 |
| 成本模式 | 订阅制 | 一次性购买 + 运维 |
| 定制能力 | 有限 | 完全自由 |

在 AstraSchedule 中，SaaS 模式下数据通过 namespace 隔离，每个学校有独立的数据空间。数据安全合规要求高的学校建议选择独立部署。
