# MySQL

## 通俗解释

MySQL 是一个关系型数据库管理系统，需要单独安装和运行。适合数据量大、并发高的场景。很多网站和应用都用 MySQL 存储数据。

AstraSchedule 支持 MySQL，适合多学校共用或高并发场景。

## 专业解释

MySQL 是一个开源的关系型数据库管理系统（RDBMS），使用 SQL 语言操作数据。

特点：
- **成熟稳定**：20多年历史，社区活跃
- **高性能**：支持大规模并发读写
- **可扩展**：支持主从复制、分库分表
- **生态丰富**：大量工具和第三方库

与 SQLite 对比：
| 特性 | SQLite | MySQL |
|------|--------|-------|
| 部署 | 单文件 | 需要服务器 |
| 并发 | 单写多读 | 多写多读 |
| 扩展性 | 单机 | 支持集群 |
| 适合场景 | 小型应用 | 大型应用 |

在 AstraSchedule 中，MySQL 适合多学校部署，配置在 `config.toml` 的 `[database]` 部分。
