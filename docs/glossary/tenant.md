# 租户（Tenant）

## 通俗解释

租户就是"共用一套系统的独立用户"。比如一栋楼里住了不同的人，每家人有自己的房间，但共用电梯和水管。在 AstraSchedule 中，大家用同一套系统，每个子域名代表一个租户，而数据互相看不到。

## 专业解释

租户是多租户架构中的核心概念，指共享同一应用实例的独立组织或用户群体。

多租户架构特点：
- **数据隔离**：每个租户的数据独立存储，互不可见
- **共享基础设施**：多个租户共用服务器、数据库等资源
- **统一部署**：一套代码服务所有租户
- **租户识别**：通过 namespace、tenant_id 等字段区分

在 AstraSchedule SaaS 模式中：
- 每个学校是一个租户
- 租户通过 namespace 标识（如 `cn/getastra/myschool-do`）
- 所有数据表包含 `namespace` 字段
- API 请求自动过滤当前租户数据

租户与用户的区别：
- **租户**：组织级别（如学校）
- **用户**：个人级别（如管理员、老师）
