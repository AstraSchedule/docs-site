# 安装客户端

客户端安装于各班电脑上，负责数据的展示。

## 下载

请前往 [GitHub Releases](https://github.com/daizihan233/AstraSchedule/releases/latest) 下载客户端本体。按提示步骤安装。

在任务栏中单击软件图标调出菜单，再点击菜单中的“云端服务”按钮，输入服务端地址，确认后重启应用即可。

> [!TIP] 
>
> 以下写法均是正确的：
>
> ```
> class.khbit.cn
> ```
>
> ```
> class.khbit.cn/prefix
> ```

![image-20260523180231776](https://image-hk-1.oss-accelerate.aliyuncs.com/image-20260523180231776.png)

如果您的后端同时部署了 TLS，那么您应该保持“安全连接”的开启；反之，则应该关闭此选项，否则会由于安全校验不通过而无法正常使用。

## 常见错误

### 末尾多一个斜杠

比如：`class.khbit.cn/`

这会导致客户端内部处理请求时，错误地把链接拼接为 `https://class.khbit.cn//39/2023/1`，这会导致请求体不规范。这是一种未定义行为，不保证在这种情况下服务端如何处理该请求，也不保证这种情况下可能出现的问题。

### 开头多一个协议头

比如：`https://class.khbit.cn`

应用本身提供了一个选项允许你选择使用 TLS 加密还是直连，应该通过在菜单内勾选“安全连接”选项来进行调整，绝对不能自己添加协议头。与前者相同，这会导致客户端内部生成错误的请求体，实际请求的是 `https://https://class.khbit.cn`，这是一个**错误的**地址，会导致请求无法到达，表现为客户端持续处于“加载中”状态，服务端无任何日志输出。
