# 安装指南

开始之前，先看看哪种部署方式更适合你。

## 快速对照

|  | 🚀 PaaS 部署（推荐） | 🖥️ 传统部署 |
|---|---|---|
| **一句话** | 后端跑在阿里云函数计算，管理端托管到 Netlify，注册账号就能用 | 用自己的服务器跑后端和管理端 |
| **适合谁** | 大多数学校，想省事省钱 | 有闲置服务器、部署在学校内网 |
| **需要服务器** | ❌ 不需要 | ✅ 一台 Linux 或 Windows 服务器 |
| **部署位置** | 公网（阿里云 + Netlify） | 学校内网 |
| **月成本** | ≈ 0 元（函数计算免费额度内） | 服务器电费 |
| **安全维护** | 必须配 Cloudflare CDN（免费 WAF，防恶意调用） | 内网无需 WAF |
| **前提条件** | 有域名，愿意花十分钟配 Cloudflare | 有能 24 小时开机的设备 |
| **上手难度** | ★☆☆ | ★★★ |

## 帮我选

回答几个问题，帮你找到最合适的指南。

---

### 第一问：你平时用过命令行吗？

| 你的情况 | 建议 |
|----------|------|
| 没用过，也不打算学 | → 📘 [PaaS 保姆级指南](./paas/guide.md) |
| 会一点基本的 | → 继续第二问 |
| 日常工作就在用 | → 继续第二问 |

---

### 第二问：你有设备可以当服务器吗？

| 你的情况 | 建议 |
|----------|------|
| 学校有台闲置电脑或服务器，能 24 小时开机 | → 继续第三问 |
| 什么都没有 | → 继续第四问 |

---

### 第三问：你打算让服务在哪里跑？

| 你的情况 | 建议 |
|----------|------|
| 就放学校内网，不对外 | → 🖥️ 传统部署（有内网服务器的话） |
| 校内校外都要能访问 | → 继续第四问 |

---

### 第四问：你有属于自己的域名吗？

PaaS 部署必须绑定域名。没有的话可以花几十块注册一个（.top、.xyz 等很便宜）。

| 你的情况 | 建议 |
|----------|------|
| 有域名 | → 继续第五问 |
| 没有，但愿意花几十块注册一个 | → 继续第五问 |
| 不想搞域名 | → 🖥️ 传统部署（内网直接用 IP 访问） |

---

### 第五问：你愿意给域名配 Cloudflare CDN 吗？

PaaS 部署在公网上，API 接口会暴露到互联网。**必须配置 WAF** 来阻止恶意调用，否则可能产生意外费用。推荐用 Cloudflare 的免费 CDN（自带 WAF），几分钟就能配好。

| 你的情况 | 建议 |
|----------|------|
| 愿意，几分钟就能配好 | → 🚀 PaaS |
| 不太懂，但愿意照教程操作 | → 📘 [PaaS 保姆级指南](./paas/guide.md)（手把手教） |
| 不想操心这些 | → 🖥️ 传统部署（内网天然隔离，无需 WAF） |

---

### 第六问：预算呢？

| 你的情况 | 建议 |
|----------|------|
| 能不花钱就不花钱 | → 🚀 PaaS（免费额度够用，域名几十块一年） |
| 可以接受小额开销 | → 🚀 PaaS 或 🖥️ 传统都行 |

---

> 💡 **大部分人的结论是 PaaS**。有域名、愿意花十分钟套 Cloudflare CDN → 零运维、零服务器成本。否则传统部署跑内网，省去域名和 WAF 的麻烦。
