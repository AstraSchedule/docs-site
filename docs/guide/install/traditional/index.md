# 传统部署

适合有自己的服务器、部署在学校内网的场景。

## 架构图

<!-- TODO: 插入架构图 -->

整体流程：客户端/管理员浏览器 → 内网服务器（后端 + 管理端）。所有服务跑在同一台机器上，不暴露到公网。

## 前提条件

- 一台能 24 小时开机的设备（Windows 或 Linux）
- 设备在学校内网，有固定 IP（或能 DHCP 绑定）
- 会基本的命令行操作

## 为什么选传统部署？

- 学校内网不需要操心 WAF 和域名
- 所有数据都在本地，完全自控
- 不依赖任何云平台
