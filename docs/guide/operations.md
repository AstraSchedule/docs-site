# 日常运维

## 常用操作

### 后端

```shell
# 查看后端状态（传统部署、systemd 管理）
systemctl status astrago

# 重启后端
systemctl restart astrago

# 查看日志
journalctl -u astrago -f
```

PaaS 部署在阿里云函数计算控制台查看日志和监控。

### 管理端

修改前端代码后需重新构建：

```shell
cd /opt/astra-web
git pull                    # 拉取最新代码
npm run build               # 重新构建
```

Netlify 部署会自动检测 GitHub 仓库更新并自动构建，无需手动操作。

### 客户端

- 托盘右键 →「检查更新」可手动更新客户端
- 托盘右键 →「刷新天气」强制刷新天气数据
- 托盘右键 →「更新课表」立即从服务端拉取最新配置

## 备份

**建议定期备份**，尤其是学期初录入大量数据后。

### 管理端备份

进入管理端 →「实用工具」→「完整导出」，下载 JSON 文件保存。

### 手动备份（传统部署）

```shell
# SQLite 数据库文件直接复制即可
cp /opt/astra/astra.db /backup/astra-backup.db
```

### 恢复

进入管理端 →「实用工具」→「完整导入」，选择备份文件上传。支持跨数据库类型迁移（MySQL ↔ SQLite）。

## 监控

### 后端健康

```shell
# 检查后端是否正常运行
curl http://192.168.1.100:9000/web/menu
```

正常返回 JSON 菜单数据。返回空或连接拒绝说明后端异常。

### 客户端连接状态

进入管理端首页，可看到所有客户端的 WebSocket 连接状态和今日异常断连次数。

### 告警信号

| 现象 | 可能原因 |
|------|----------|
| 管理端白屏或一直加载 | 后端挂了，或 `global.js` 中地址配错 |
| 客户端显示离线配置 | 客户端连不上后端，检查网络和服务端状态 |
| 页面显示"无作息/无课程" | 该班级配置不完整，进管理端补齐 |
| 天气不显示 | 和风天气 API 配额用完，或 API Key 配置有误 |
| 自动任务没生效 | 检查任务状态是否为"待生效"且日期范围正确 |

## 发布流程（开发者）

1. 在测试环境验证配置读写
2. 发布到生产环境
3. 关注以下接口健康：
   - `/web/statistic` — 统计概览
   - `/web/menu` — 菜单结构
   - `/web/config/*` — 各配置接口
