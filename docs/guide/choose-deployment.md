# 如何选择部署方案

AstraSchedule 提供四种部署方案，根据你的实际情况选择。

## 快速决策

| 你的情况 | 推荐方案 |
|----------|----------|
| 不想折腾服务器，先用起来 | [SaaS 免部署](./saas) |
| 有一台服务器，学校有内网 | [学校内网部署](./intranet/) |
| 没有服务器，想低成本外网访问 | [极低成本外网部署](./lowcost/) |
| 学校规模大，访问量高 | [外网高并发部署](./production/) |
| 学校有数据安全合规要求 | 选任意一种**独立部署**方案 |

## 四种方案对比

| | SaaS | 极低成本 | 外网高并发 | 学校内网 |
|---|---|---|---|---|
| 需要服务器 | 不需要 | 不需要 | 需要 | 需要 |
| 需要域名 | 不需要 | 需要 | 需要 | 不需要（内网 IP） |
| 费用 | 目前免费 | 比较便宜   | 较高 | 不一定，需要具体情况具体分析 |
| 技术门槛 | 低 | 中 | 高 | 高 |
| 访问速度 | 快 | 快 | 快 | 最快 |
| 适合规模 | 任意 | < 100 个班 | > 100 个班 | 单校 |
| 数据控制 | 托管 | 自己管理 | 自己管理 | 自己管理 |

## 按需求选择

### 我不想自己维护服务器

选 **SaaS 免部署**。注册账号就能用，服务器、数据库、HTTPS 都不用管。缺点是不能自定义域名，数据存在我们这边。

### 我想让学生在家也能看课表

选 **极低成本外网部署**。用阿里云函数计算 + Cloudflare，一年不到 100 元。缺点是部署和维护比较麻烦。

### 我要给几十所学校用

选 **外网高并发部署**。用 MySQL + 阿里云函数计算 + ESA，支持大规模并发。缺点是成本较高，需要一定运维能力。

### 我学校有内网，学生不出校门

选 **学校内网部署**。服务器放学校机房，学生在校内直接访问，速度快、安全。缺点是校外访问不了，而且需要运维人员具有传统服务器的运维技能。

### 我学校有数据安全合规要求，必须有数据的所有权

选任意一种**独立部署**方案（极低成本、外网高并发、学校内网）。SaaS 模式下数据存在我们这边，无法满足数据主权要求。独立部署的数据完全由你控制，可以放在学校机房或你自己指定的云服务器上。

但是这 **并不意味着** 独立部署的数据就更加安全，独立部署需要自行配置安全防护，倘若配置不当，安全性可能远不及 SaaS。SaaS 已内置的安全配置已经足够抵挡绝大部分网络攻击，对大多数学校来说也更省心。

除非您有确切的数据主权要求，否则我们不建议您使用独立部署，因为配置不当很可能使数据更加危险。

## 技术细节对比

| | SaaS | 极低成本 | 外网高并发 | 学校内网 |
|---|---|---|---|---|
| 后端 | 函数计算 | 函数计算 | 函数计算 | 自建服务器 |
| 数据库 | SQLite | SQLite | MySQL | SQLite |
| WebSocket | 不支持（轮询） | 不支持（轮询） | 不支持（轮询） | 支持 |
| HTTPS | Cloudflare | Cloudflare | ESA | 通常不需要 |
| WAF | Cloudflare | Cloudflare | ESA | 通常不需要 |
| 备份 | 自动（阿里云云备份） | 手动 | RDS 自动 | 手动 |

可以简单地认为，SaaS 就是一个已经部署好了的极低成本部署版本，这也是 SaaS 免费的最主要原因。

WebSocket 并 **不是** 一个很重要的功能，实际应用中其发挥作用的次数很少，此功能也只是为了兼容旧版本客户端而被保留，故不建议因此盲目选择内网部署。

> [!TIP]
>
> SaaS 与极低成本部署版本仍存在细微区别，是两个维护分支，各自受长期维护，且不会合并。

## 迁移路径

各方案之间可以迁移：

1. **SaaS → 独立部署**：导出 JSON 备份，导入新后端
2. **SQLite → MySQL**：导出 JSON，切换数据库类型，导入
3. **内网 → 外网**：配置域名和 HTTPS，调整防火墙

建议：先用 SaaS 或 SQLite 试用，确认需求后再决定是否迁移到更复杂的方案。
