# 常见问题

## 配置相关

### Q1：为什么作息保存后 divider 会变化？

系统会自动做 key 一致性修正：

- 缺失的 divider[key] 自动补 []
- 多余的 divider[key] 自动移除

这是为了避免前端渲染与后端计算时的数据歧义，属正常行为。

### Q2：复制配置会复制哪些内容？

科目、作息、课表、通用设置。

### Q3：复制配置失败提示来源不存在？

检查来源学校/年级/班级是否已配置完整，尤其是班级级别的课表与通用设置。

### Q4：是否支持只复制一部分配置？

当前默认复制整套配置。若需要细粒度复制，可用备份导出后手动拼接 JSON。

### Q5：常日 作息为什么删不掉？

常日 是系统兜底作息。当某天没有匹配的作息表时，系统会回退到 常日，因此不可删除。

## 部署相关

### Q6：客户端连不上后端怎么办？

1. 确认后端已启动（curl 测试一下）
2. 确认客户端填的地址正确：
   - 内网：192.168.1.100:9000 或 http://192.168.1.100:9000
   - PaaS：pi.your-domain.com
3. 内网环境检查防火墙是否放行了 9000 端口
4. PaaS 环境检查 Cloudflare DNS 代理状态是否为 🟢

### Q7：PaaS 部署收到大量 403 怎么办？

检查 Cloudflare WAF 规则，可能是误拦截。在 WAF 日志中查看具体规则，将误判规则设为「跳过」。

### Q8：函数计算冷启动很慢？

首次请求或长时间无请求后，函数计算需要冷启动。通常几秒内完成。如果介意延迟，可在函数计算控制台配置「预留实例」来保持一个实例常驻（会产生少量费用）。

### Q9：SQLite 和 MySQL 怎么选？

- **SQLite**：无需安装数据库，数据存文件，适合小规模使用（几个班级），备份就是复制文件
- **MySQL**：适合多个学校共用、高并发场景。需要单独安装和配置

如果拿不准，先用 SQLite，以后可以通过备份导入功能迁移到 MySQL。

## 客户端相关

### Q10：客户端窗口遮挡了其他内容怎么办？

- 鼠标悬停在窗口上 → 窗口透明度恢复正常
- 鼠标移开 → 窗口降至几乎透明
- 托盘菜单中可开关「窗口置顶」「点击穿透」

### Q11：为什么天气不显示？

1. 检查 config.toml 中 pikey 是否配置正确
2. 检查客户端选择的地区是否正确（托盘菜单 → 当前地区）
3. 和风天气免费 API 有日调用量限制（API Key 方式 1000 次/天，JWT 方式无限）

### Q12：倒数日窗口不显示？

1. 确保管理端已配置倒数日
2. 检查倒数日配置的「生效域」是否覆盖了当前班级
3. 过期的倒数日会自动隐藏

### Q13：自动更新失败？

1. 检查托盘菜单 → 更新源设置是否正确
2. 如果网络无法访问 GitHub，切换为镜像源
3. Windows 7 系统可能需要安装特定补丁才能正常更新

### Q14：课表上科目名称显示不对？

检查「科目管理」中简称和全名的映射关系。简称支持 @ 下角标语法（如 自@物 → 自ᵥ）。
