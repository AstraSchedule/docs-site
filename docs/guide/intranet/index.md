# 学校内网部署

> 🏫 适合有闲置服务器，想把所有服务放在内网的学校。

## 架构

<!-- TODO: 架构图 -->

```
客户端/浏览器 → 学校内网服务器 (后端 + 管理端)
```

所有服务跑在同一台内网机器上，不暴露到公网。

## 方案说明

| 组件 | 选型 | 说明 |
|------|------|------|
| 服务器 | 自建（Linux 推荐） | 一台能 24 小时开机的电脑即可 |
| 后端 | Go 二进制直接运行 | systemd 管理，开机自启 |
| 数据库 | SQLite（推荐）或 MySQL | SQLite 零安装，备份即复制文件 |
| 管理端 | Nginx 静态托管 | 构建后放在 Nginx 下 |
| 安全防护 | 内网隔离 | 不暴露到公网，无需额外防护 |

## 你需要准备

- 一台能 24 小时开机的设备（Windows/Linux）
- 固定内网 IP（联系学校网管或 DHCP 绑定）

## 部署步骤

1. [安装后端](./server.md) — 下载二进制，配置 systemd
2. [安装管理端](./web.md) — 构建前端，配置 Nginx
3. [安装客户端](./client.md) — 教室电脑安装配置
